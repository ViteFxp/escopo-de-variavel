Escopo de Variáveis em JavaScript (Wiki - Dark Mode)

Em programação, o escopo define a região do código onde uma variável é visível e acessível. Esta página explica e demonstra as diferenças entre Escopo Global e Local.

Tipo 1: Variáveis Globais 🌍

Definidas no escopo principal, fora de qualquer função. São amplamente acessíveis.

  • **Acessibilidade:** Podem ser lidas de qualquer lugar do código.
  • **Ciclo de Vida:** Existem durante toda a execução do script.

Tipo 2: Variáveis Locais 🏠

Definidas dentro de uma função ou bloco de código. Sua visibilidade é restrita.

  • **Acessibilidade:** Apenas dentro da função ou bloco onde foram declaradas.
  • **Ciclo de Vida:** São destruídas após a execução do seu escopo.

Demonstração Visual (JavaScript)

O script abaixo demonstra o fluxo de visibilidade no Console. Para acompanhar a simulação passo a passo, abra o console do seu navegador (F12).

INSTRUÇÃO: Use **F12** e vá para a aba **Console** para ver a saída de execução, incluindo o erro de escopo!


// ======================================================================
// CÓDIGO DEMONSTRATIVO NO CONSOLE
// ======================================================================

// VARIÁVEL GLOBAL
let VAR_GLOBAL = "Eu sou GLOBAL (Escopo Principal)";
console.log(`1. FORA DA FUNÇÃO (GLOBAL): ${VAR_GLOBAL}`);

function demonstrarEscopo() {
    // VARIÁVEL LOCAL
    let VAR_LOCAL = "Eu sou LOCAL (Dentro da função)";
    
    console.log("\n2. DENTRO DA FUNÇÃO:");
    console.log(`   Acessando VAR_LOCAL: ${VAR_LOCAL}`);
    console.log(`   Acessando VAR_GLOBAL (Leitura): ${VAR_GLOBAL}`);
    
    // Mascaramento: Cria uma nova variável LOCAL chamada VAR_GLOBAL. 
    // A global original não é afetada.
    let VAR_GLOBAL_LOCAL = "AGORA sou LOCAL e Mascarei a Global";
    console.log(`   Nova VAR_GLOBAL (Local) ATRIBUÍDA: ${VAR_GLOBAL_LOCAL}`);
}

demonstrarEscopo();

// Tentativa de acesso à Local: FALHA
console.log("\n3. FORA DA FUNÇÃO (Tentativa de acesso à Local):");
try {
    // Este comando resultará em um ReferenceError
    console.log(`   Tentando acessar VAR_LOCAL: ${VAR_LOCAL}`);
} catch (error) {
    console.error(`   ERRO CAPTURADO: ${error.name}: ${error.message}`);
    console.log("   *Isso prova que VAR_LOCAL é invisível fora de sua função.");
}

// Visualizando a Global após a função
console.log("\n4. FORA DA FUNÇÃO (Global após a função):");
console.log(`   O valor original de VAR_GLOBAL não mudou: ${VAR_GLOBAL}`);

Comparativo Rápido

Característica Variável GLOBAL 🌍 Variável LOCAL 🏠
**Definição** Fora de qualquer função/bloco. Dentro de uma função/bloco.
**Visibilidade** Todo o código. Apenas dentro da sua função/bloco.
**Ciclo de Vida** Do início ao fim do programa. Apenas durante a execução do escopo.
**Isolamento** Baixo (Alto risco de conflitos). Alto (Recomendado para boas práticas).
